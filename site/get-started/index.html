Get started

Set up agent-ssh in a few steps

The default path is straightforward: install the binary, generate a config, validate it, then run named profiles. Unrestricted sessions are there when a user explicitly enables them and the server configuration opts in.

1. Install the binary

Use whichever install path matches your environment:

brew tap aibunny/agent-ssh https://github.com/aibunny/agent-ssh
brew install agent-ssh
curl -fsSL https://raw.githubusercontent.com/aibunny/agent-ssh/main/scripts/install.sh | sh
cargo install --git https://github.com/aibunny/agent-ssh agent-ssh-cli

2. Create and validate a config

agent-ssh init
agent-ssh config validate

The generated agent-ssh.toml includes commented examples for servers, profiles, approvals, and the optional unrestricted-session flag.

3. Run a named profile

This is the normal mode. The server defines which profiles are allowed, and the caller gets the full output back. 

agent-ssh exec --server staging-api --profile disk

agent-ssh exec --server staging-api --profile logs \
  --arg service=api \
  --arg "since=10 min ago"

4. Open an unrestricted session when settings allow it

Arbitrary commands are not the default path. To use them, both of these must be true:

  • The server has allow_unrestricted_sessions = true and requires_approval = true.
  • The user settings in the agent-facing layer allow arbitrary commands.
[servers.prod-web-1]
host = "10.0.10.21"
user = "deploy"
environment = "production"
allowed_profiles = ["logs"]
requires_approval = true
allow_unrestricted_sessions = true
agent-ssh session open \
  --server prod-web-1 \
  --mode unrestricted \
  --approval CAB-1234

agent-ssh session exec \
  --session <session-id> \
  --cmd "uname -a && whoami && pwd"

The broker keeps that connection alive and multiplexes later commands over the same SSH session until it is closed or expires.
