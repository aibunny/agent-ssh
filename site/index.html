Security-first remote access

Brokered SSH that tells agents exactly what they can do.

agent-ssh lets operators define named servers and approved command profiles once, then gives humans and AI agents a clean way to run them with captured output, audit logs, and optional broker-held persistent sessions.

$ agent-ssh exec --server staging-api --profile logs \
    --arg service=api --arg "since=5 min ago"

What it does

Turns raw SSH access into a brokered interface.

  • Maps friendly aliases to real hosts so agents never need host details.
  • Runs named command profiles by default and always captures full output.
  • Supports unrestricted sessions only when the user and server settings allow them.

Why it matters

Safer than handing an agent an SSH key and hoping for the best.

The broker keeps policy checks, approvals, and audit logging in one place. Operators stay in control, and callers get a simpler interface.

How it works

From config file to broker-held session

01

Define servers and profiles

Operators configure named servers, approval rules, and safe command profiles in TOML.

02

Run a profile or open a session

Callers use the alias-based CLI or agent layer instead of crafting raw SSH invocations.

03

Capture output and audit everything

Every broker decision and remote command result is written to the audit log.

Quick install

Pick the install path that fits your environment

Homebrew

brew tap aibunny/agent-ssh https://github.com/aibunny/agent-ssh
brew install agent-ssh

One-line installer

curl -fsSL https://raw.githubusercontent.com/aibunny/agent-ssh/main/scripts/install.sh | sh

Cargo

cargo install --git https://github.com/aibunny/agent-ssh agent-ssh-cli
