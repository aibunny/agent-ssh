Commands, configuration, and session controls

Core commands

agent-ssh init

Create a starter TOML config in the current directory.

agent-ssh config validate

Parse and validate the config before attempting any SSH activity.

agent-ssh exec

Run a named profile, capture output, and mirror the remote exit code.

agent-ssh session open

Open a restricted or unrestricted broker-held session.

agent-ssh session exec

Execute a named profile or raw command inside an open session.

agent-ssh session close

Close the broker-held SSH connection and clean up the session record.

Top-level config sections

  • [broker] for certificate TTL, audit log path, and default signer.
  • [signers.<name>] for signer definitions.
  • [servers.<alias>] for named SSH targets and policy.
  • [profiles.<name>] for reusable approved commands.
[servers.prod-web-1]
host = "10.0.10.21"
user = "deploy"
environment = "production"
allowed_profiles = ["logs"]
requires_approval = true
allow_unrestricted_sessions = true

allow_unrestricted_sessions = true is the server-side opt-in for arbitrary commands. It only matters when approvals are also required.

Session behavior

  • Restricted sessions use named profiles.
  • Unrestricted sessions accept raw --cmd input.
  • Default TTL is 300 seconds, with a max of 3600 seconds.
  • Default idle timeout is 60 seconds.
  • Persistent sessions are not supported for legacy password auth.
